SOCIETIES Virgo Node Configurations

The SOCIETIES Virgo node configurations are as follows:
• Full Container/Combined Node - Includes one domain authority and one user container node
• User Container
• Domain Authority (DA) Container - (at least one DA node is required for each platform deployment)

The default configuration (as of Release 1.0) is a multi-user container configuration of one combined node (Domain Authority + CSS user container) and one CSS user container.

The combined node container uses the default Virgo directories, i.e. VIRGO_HOME/config and VIRGO_HOME/pickup. All additional user containers use a custom directory structure, e.g. VIRGO_HOME/config2 and VIRGO_HOME/pickup2. The platform properties for each container are configured in the respective properties file, e.g. VIRGO_HOME/config/org.societies.platform.properties.

For more information on configuring multiple nodes and container port configurations please read:
• Configuration for supporting more users with MultipleContainers https://redmine.ict-societies.eu/projects/sp/wiki/MultipleContainers
• Running the DomainAuthority Node https://redmine.ict-societies.eu/projects/sp/wiki/DomainAuthority
