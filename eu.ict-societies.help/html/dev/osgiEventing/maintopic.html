Introduction

The eventing and pubsub feature provided by the SOCIETIES Platform allows 3rd party service developers to exploit those designs patterns within their services. There are currently 2 eventing models available.


1. Internal eventing between bundles within the container using the OSGI EventAdmin. Events only available to bundles running within the same container. Supports native java objects as payloads
2. External eventing between conecting clients (including other containers). Allows a CSS Node to create an event stream for other clients (CSS nodes) to subscribe to. Supports schema generated objects only as payload.

The Pubsub service is dependent on the communication framework. Please ensure you have deployed the Comms Frwk bundles first.
