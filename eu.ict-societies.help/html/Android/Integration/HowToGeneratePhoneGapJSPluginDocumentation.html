Android Javascript (JS) API¶

As Javascript is a dynamic, weakly typed language which uses Prototypical inheritance rather the classical OO inheritance model, it can be difficult for developers to determine an API from a JS source(s). Fortunately, the JSDOC project provides a set of annotations with which to document JS code in a similar way to the JavaDoc.

Since PhoneGap uses the plugin model to extend its JS API, any developer creating a native Societies core component will have to also provide a PhoneGap plugin (Java class) and a complementary set of JS functions to allow their native component's functionality to be accessed by JS in order to be controlled and displayed to users via a HTML web view. The MasterGUI project already has some plugins and JS functions which act as the native components' JS API. In order to generate the current JS API carry out the following steps:

1. Locate the MasterGUI project either by command line or by Eclipse.

2. $ mvn site

3. The generated JS API documentation will be available at:

SOCIETIES-Platform/platform-infrastructure/client-frwk/MasterGUI/target/doc

4. Locate the above location using a browser a select the "index.html" file.
