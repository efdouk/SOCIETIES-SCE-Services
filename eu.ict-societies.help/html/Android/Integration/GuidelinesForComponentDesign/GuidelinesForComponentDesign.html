Design Decisions¶

The various Android Societies components will require some GUI/user interaction. For example, CSS registration, activity feeds, User interaction Monitor etc. Rather than having each component create its own GUI activity(s), it is proposed that a single GUI will be created to cover all aspects of user interaction with the Societies Core Services (WP4) and the various WP5 functionalities layered on top. This design decision will allow the the following:

  • A single integrated GUI to control and allow user interaction with the Societies Android client
  • Encourage a consistent user experience
  • Allow steamlined user interaction flows to be developed
  • Encourage consistent user notifications and dialogs

The decision has also been made to use Web-based activities rather than native-based activities in conjunction with PhoneGap and JQueryMobile. This will mean that the screens will be composed of HTML5+CSS with JQueryMobile attributes and the use of Javascript to add and control dynamic content. Some benefits that flow from this set of tools are:

  • Allow for the porting of the Societies Light Client to other potential mobile operating systems and a future date leveraging PhoneGap
  • JQueryMobile will provide a lot of the page navigation, AJAX and user events functionality along with page animation effects, theming and cross browser consistency
  • PhoneGap provides an existing Javascript API to access device resources such as addresss books, sensors and connectivity with an extensible plugin architecture
  • Non-programming web designer personnel can work on the HTML+CSS aspects of the client
  • Developers can concentrate on the Javascript and Javacsript-Native PhoneGap plugins (Java classes) to extend the PhoneGap API and provide additional Societies functionality
  • The HTML+CSS+Javascript code base can be used for non-Android browser-based development with minimal changes

Running the GUI application¶

There are three ways of getting a Android GUI application working:
  • Browser: As the content is HTML based, point any browser at your local Git repository using a localised version of the following URI: "file:///<local path>/SOCIETIES-Platform/platform-infrastructure/client-frwk/MasterGUI/assets/www/index.html". The main page of the application should appear.
  • Android Emulator: This option will require the installation of the Android SDK, the creation of an Android Virtual Device of the name "Test22Device" and supporting Android 2.2. Compile the MasterGui project using Maven or even easier, Eclipse, start the relevant AVD, deploy the application and run the app. The running of the application can be slow and HTML pages look longer than expected.
  • Android Device: This option is similar to the above option except that the application is deployed and run on an actual Android device that is at least Android 2.2. This option is the best for reviewing the final build in terms of speed, layout and whether there are any phone specific browser problems.

What is PhoneGap?¶

PhoneGap is a platform for building mobile application using Web technologies: HTML, CSS and Javascript. Mobile platforms like Android, iPhone or Blackberry are not ready yet to manage such application, that is why PhoneGap is filling in the gap by providing a plugin system to use native call through Javascript.

To resume, PhoneGap is providing:

  • a PhoneGap Builder: a way to embed an HTML, CSS, Javascript web application into a mobile application (Android, iPhone, Blackberry and others)
  • a PhoneGap API: native PhoneGap plugins (see next point) to access:
    • Javascript standard APIs not yet available on mobile platform (e.g. HTML5 WebSocket , ...)
    • well-known controls and actions on mobile device not (totally) yet defined as standards (e.g. access the contact list, make a call, ...)
  • a PhoneGap Plugin system to enlarge Javascript possibilities by using native call
    • a set of PhoneGap plugin is already available: PhoneGap plugins list

PhoneGap is an open source project recently lead by Adobe and incubated by the Apache Cordova project . “PhoneGap” may be renamed “Cordova” in a near future.
