Tips to Manage the JSON Format¶

The purpose of this tutorial is to learn some best practices to use more easily the JSON format. JSON is a string format used very often in Web technologies and particularly with Javascript because it is so easy to manipulate this format in this language (it has been created for that). This format is used in the light client to use native calls (e.g. Android calls) through Javascript on the GUI component.

JSON Format and Javascript¶

"JSON" means "Javascript Object Notation". So, the JSON format is very easy to use in Javascript. As you can see in the JSON website (Page dedicated to Javascript on the JSON website), a lot of objects simplify the use of JSON in Javascript:
  • JSON.parse(string) to convert a string into a JSON object
  • eventually eval('('+string+')') to convert a string into a JSON object, but it is not safe, the previous method is better
  • JSON.stringify(json object) to convert a JSON object into a string

JSON Format and Java (and Android)¶

JSON is natively supported by Java. As you can see in the JSON website (Page dedicated to Java on the JSON website), a lot of objects simplify the use of JSON in Java. . Other libraries may help to manipulate this format. We propose to follow best practices bellow:

  • Use the native library: to simple manipulation only. E.g. fill a JSON object with values... JSONObject and JSONArray are usually used.
  • Use GSON library: to easily create a JSON object from a Java object (toJson(java object)), or fill a Java object from a JSON object (fromJson(json string)).
    • GSON user guide
    • How to Use GSON (tutorial for beginners)
  • Or maybe use the Jackson library: equivalent to GSON, but a little bit more complicated to use for the same performance. Proposition: use GSON.
