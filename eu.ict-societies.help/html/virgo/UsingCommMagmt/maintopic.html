This section provides guidelines regarding the use of the SOCIETIES CIS Manager component by third party and internal services. The CIS Manager Cloud is the component used by the user to manage his own CISs and the CISs in which he is part of.

Dependencies (pom.xml)

Add the following dependencies to your pom.xml file:

1 <dependency> 2 <groupId>org.societies.api.external</groupId> 3 <artifactId>societies-api</artifactId> 4 <version>${release.version}</version> 5 <scope>provided</scope> 6 </dependency>

SGi service configuration (bundle-context-osgi.xml)

<osgi:reference id="CisManagementBundle" interface="org.societies.api.cis.management.ICisManager"/>

OpenFire Configuration

The CIS Manager connects to the XMPP server through the XCComunication Manager. That component must be configured with a properties file such as this one and so that the secretKey matches with the "Default shared secret" in OpenFire (Service Enabled box under Server Settings - External Component Settings) allowing any component to connect to it.
